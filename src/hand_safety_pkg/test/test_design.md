# Hand Safety Node 测试场景设计

## 当前安全规则概览

| 规则 | 描述 |
|------|------|
| Rule 1 | 连续三帧中第 4/5/6 自由度大于等于 0 且小于等于 200 → 钳位到 200 |
| Rule 2 | 任意自由度受力（state.force_act）> 3000 → 用上一帧覆盖 |
| Rule 3 | 任意自由度指令超出 0~1000 → 裁剪到边界 |
| Rule 4 | 连续五帧相邻帧变化 > 200 → 取五帧平均值 |

---

## 一、单规则测试

### Rule 1（第 4/5/6 自由度持续低位 → 钳位 200）

| 编号 | 场景 | 预期结果 |
|------|------|----------|
| 1.1 | 连续 3 帧，joint 3/4/5 全部为 100 | 触发 Rule 1，三个关节输出 200，其余不变 |
| 1.2 | 连续 3 帧，joint 3/4/5 全部为 200（上边界） | 触发 Rule 1 |
| 1.3 | 连续 3 帧，joint 3/4/5 全部为 0（下边界） | 触发 Rule 1 |
| 1.4 | 连续 3 帧，joint 3/4/5 全部为 -1 | **不触发**（条件是大于等于 0） |
| 1.5 | 连续 3 帧，joint 3/4/5 全部为 201 | **不触发**（条件是小于等于 200） |
| 1.6 | 只有 2 帧满足（队列还没满 3 帧） | **不触发** |
| 1.7 | 前 2 帧满足，第 3 帧 joint 2 = 250 | **不触发** |
| 1.8 | 连续 3 帧，joint 3/4 满足但 joint 5 = 300 | **不触发**（三个必须同时满足） |
| 1.9 | 连续 3 帧，joint 3/4/5 有负值（如 -50） | **不触发**（不满足 >= 0） |

### Rule 2（受力过大 → 用上一帧覆盖）

| 编号 | 场景 | 预期结果 |
|------|------|----------|
| 2.1 | state 反馈 force_act 某个关节 = 3001 | 触发 Rule 2，该关节用上一帧值覆盖 |
| 2.2 | state 反馈 force_act 全部 6 个关节 > 3000 | 触发 Rule 2，所有关节全部被覆盖 |
| 2.3 | state 反馈 force_act 某个关节 = 3000（边界值） | **不触发**（条件是严格大于 3000） |
| 2.4 | **没有收到 state 消息**（latest_state 为 None） | 降级使用 msg.force_set 判断 |
| 2.5 | state 消息的 force_act 为空列表 | 降级使用 msg.force_set 判断 |
| 2.6 | 第一帧就触发（没有历史），force_act > 3000 | 使用 fallback 默认帧（angle_set 全 1000）覆盖 |
| 2.7 | force_act 长度不足 6（如只有 4 个元素） | 只检测前 4 个关节，后 2 个不受影响 |

### Rule 3（指令越界 → 裁剪到 0~1000）

| 编号 | 场景 | 预期结果 |
|------|------|----------|
| 3.1 | 某关节 angle_set = -100 | 裁剪为 0 |
| 3.2 | 某关节 angle_set = 1500 | 裁剪为 1000 |
| 3.3 | 多个关节同时越界（如 joint 0 = -50, joint 2 = 2000） | 各自独立裁剪 |
| 3.4 | 某关节 angle_set = 0（边界值） | **不触发**（0 在范围内） |
| 3.5 | 某关节 angle_set = 1000（边界值） | **不触发**（1000 在范围内） |
| 3.6 | 极端负值 angle_set = -32768（int16 最小值） | 裁剪为 0 |
| 3.7 | 极端正值 angle_set = 32767（int16 最大值） | 裁剪为 1000 |

### Rule 4（相邻帧突变 → 五帧平均）

| 编号 | 场景 | 预期结果 |
|------|------|----------|
| 4.1 | 5 帧中 joint 0：500→500→500→500→800（差值 300 > 200） | 触发，取五帧平均 |
| 4.2 | 5 帧中 joint 0：500→700→500→700→500（差值 200） | **不触发**（差值等于 200，不严格大于） |
| 4.3 | 5 帧中 joint 0：500→701→500→701→500（差值 201 > 200） | 触发 |
| 4.4 | 5 帧中，只有第 1→2 帧差值 > 200，后面帧平稳 | 触发（只要任意相邻帧超标就触发） |
| 4.5 | 只有 3 帧，存在突变 | **不触发**（需要 5 帧满） |
| 4.6 | 所有 6 个关节同时突变 | 全部 6 个关节分别触发取平均 |
| 4.7 | 突变关节被 Rule 1 先修改过（值已被钳位到 200） | Rule 4 基于修改后的 frames_angles 判断 |

## 二、规则交叉 / 组合测试

| 编号 | 场景 | 关注点 |
|------|------|--------|
| C1 | Rule 1 钳位 joint 3/4/5 到 200，但 force_act 中这几个关节恰好 > 3000 | Rule 2 会把 Rule 1 刚设定的 200 再覆盖成上一帧的值 |
| C2 | Rule 2 覆盖后的值恰好 > 1000 或 < 0 | Rule 3 会进一步裁剪 |
| C3 | Rule 1 + Rule 4 同时作用于 joint 3/4/5 | Rule 1 先钳位到 200，Rule 4 再取平均可能把 200 拉回去 |
| C4 | 某关节被 Rule 3 裁剪后，导致与上一帧差值 > 200 | Rule 4 可能把该关节再次修正为五帧平均 |
| C5 | 全部 4 条规则同时触发 | 验证规则的执行顺序（1→2→3→4）是否正确 |

## 三、系统边界 / 异常输入测试

| 编号 | 场景 | 关注点 |
|------|------|--------|
| S1 | angle_set 长度不足 6（如只传了 3 个元素） | Rule 1 的 `f[3]`、`f[4]`、`f[5]` 会不会越界报错 |
| S2 | angle_set 为空列表 | 各规则遍历是否安全 |
| S3 | 节点刚启动，第 1 帧就是异常指令 | fallback 默认帧是否生效 |
| S4 | 左右手独立性：左手触发规则，右手正常 | 左右手的 history、latest_state 是否互不干扰 |
| S5 | 高频发送（如 200Hz）连续正常指令 | deque(maxlen=5) 的滑窗是否正确滚动 |
| S6 | state 消息比 ctrl 消息来得晚（时间错位） | 安全节点使用的是上一次的 state，判断是否仍然合理 |
| S7 | state 消息突然中断（驱动崩溃） | latest_state 停留在最后一帧旧值，是否会持续误触发 Rule 2 |
| S8 | 连续发送完全相同的正常帧 | 不应触发任何规则，指令原样透传 |
